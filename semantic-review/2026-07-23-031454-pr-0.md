## Archive tolerance stops at the envelope boundary

Non-2xx responses and malformed top-level structures remain failures, while search documents and file records decode independently. The callback transport links coroutine cancellation to `Call.cancel()`; the MockWebServer `NO_RESPONSE` regression test confirms cancellation drains OkHttp's running-call count.

## Creative Commons recognition is an explicit allowlist

Records enter the domain only when the host is exactly `creativecommons.org` and the path matches a recognized Creative Commons code/version or `publicdomain/zero|mark/1.0`. User info, custom ports, queries, fragments, deceptive hosts, and unrelated same-host paths are rejected; accepted HTTP links are canonicalized to HTTPS.

## Local-first fallback retains failure semantics

`CatalogResult` carries the current stable-order movie list and accumulated `CatalogFailure` values. `detailsResult` applies the same distinction to detail lookup, so missing content is no longer conflated with source failure; data-only compatibility methods remain available.

## SharedPreferences restoration isolates corrupt entries

Favorites and progress update synchronously before `apply()` persistence. Progress decodes one entry at a time, discarding malformed records without clearing valid resume positions and filtering blank identifiers or negative values on restoration.
