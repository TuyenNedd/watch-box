# Archive-backed catalog and resilient local-first repository (v2)

This follow-up adds the missing Internet Archive `CatalogSource`, moves OkHttp onto a cancellable callback bridge, publishes curated repository results before delayed remote enrichment, preserves coroutine cancellation, and restores progress entries independently. The prior adapter, blocking-call, delayed-local-result, swallowed-cancellation, and all-or-nothing progress findings are addressed. A fresh run completed 33 tests with no failures, compiled debug Kotlin, and the four curated media URLs returned HTTP 206. Two failure-state boundaries and the Creative Commons path validation remain incomplete.

**Watch for:** **[Important, confirmed]** repository fallback still turns remote failures into successful local/empty completion; **[Important, confirmed]** malformed whole Archive payloads still become empty DTOs; **[Important, confirmed]** Creative Commons validation accepts arbitrary paths beneath trusted prefixes; **[Minor, confirmed]** the real OkHttp cancellation bridge has no regression test.

**Verdict**: NEEDS_CHANGES

## High-level view

Repository flows now publish source-by-source snapshots in stable deduplicated order. **[Important, confirmed]** They carry only lists, so an offline remote source and a successful source with no matches are behaviorally identical; detail lookup has the same ambiguity between remote failure and not found.

**[Important, confirmed]** Successful HTTP responses containing a malformed whole document are decoded as empty success, preventing retry/error state for an upstream schema or proxy-body failure.

**[Important, confirmed]** License validation pins the host to `creativecommons.org` but uses path prefixes, so nonexistent or unrelated paths under `/licenses/` and `/publicdomain/` are accepted as open licensing.

**[Minor, confirmed]** Cancellation is tested only with a fake source that throws `CancellationException`; no test cancels an in-flight `InternetArchiveClient` request and proves the underlying `Call` is aborted.

## Repository fallback still erases remote failure state

`MovieRepository.mergeUpdates` catches every non-cancellation source exception and substitutes `emptyList()`, while `details` substitutes `null`. **[Important, confirmed]** The public APIs complete successfully with exactly the same values for an offline/503 remote source, a valid empty remote response, and a missing detail. The new `IOException` behavior is observable only to direct `InternetArchiveClient`/adapter callers, not to the repository consumer that Task 5 will use for Vietnamese offline and retry states.

Preserve the local snapshot and failure information together rather than choosing between fallback and observability. A flow value or result type can carry accumulated movies plus the failed source/error category; detail resolution needs the same distinction after all sources are exhausted.

## Whole-document decode failures remain empty success

`decodeSearch` and `decodeMetadata` wrap response-root parsing in `runCatching(...).getOrDefault(emptyDto)`. **[Important, confirmed]** A 200 response containing HTML, truncated JSON, or a top-level schema mismatch is indistinguishable from a legitimate empty catalog or metadata record. Per-entry tolerance should remain, but envelope decode failure needs a typed signal that the repository can expose without discarding curated data.

The client test explicitly expects `metadata("film id/part")` to return an empty DTO for `not-json`, locking in the failure-as-empty behavior.

## Creative Commons prefix checks do not prove a recognized license

`normalizeOpenLicense` requires the exact Creative Commons host, then accepts any path beginning with `/licenses/` or `/publicdomain/`. **[Important, confirmed]** Values such as `https://creativecommons.org/licenses/not-a-license/9.9/` and `https://creativecommons.org/publicdomain/copyright/1.0/` pass and are labeled “Giấy phép mở” or “Public Domain.” This leaves the prior arbitrary-license issue open inside the trusted host.

Validate complete recognized path shapes: approved Creative Commons codes and versions under `/licenses/{code}/{version}/`, plus explicit public-domain mark/zero paths and versions. Add deceptive same-host path cases alongside the existing deceptive-host case.

## OkHttp cancellation behavior is unverified

The callback bridge registers `call.cancel()` with `invokeOnCancellation`, but the cancellation test exercises a fake repository source that throws immediately. **[Minor, confirmed]** It does not prove that cancelling a suspended MockWebServer request terminates the network call or avoids a late callback race. Add an in-flight client cancellation test using a delayed/no-response response and assert prompt job completion plus call cancellation.
