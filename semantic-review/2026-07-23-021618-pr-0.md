# Licensed movie domain, Archive catalog plumbing, and local watch library

This change adds the Tasks 2‚Äì4 domain model, Vietnamese search normalization, four-film curated catalog, Internet Archive transport/mapping, source-merging repository, and SharedPreferences-backed favorites/progress. Curated records use HTTPS media and explicit Creative Commons metadata; Archive derivatives are ranked before becoming playback sources. A fresh JVM run reports 27 tests with zero failures and debug Kotlin compiles. The main gaps are at subsystem boundaries rather than the individual happy paths.

**Watch for:** **[Important, confirmed]** Internet Archive has no production `CatalogSource` adapter; **[Important, confirmed]** synchronous HTTP can block coroutine callers and hides transport failures; **[Important, confirmed]** repository merging delays local results and suppresses cancellation; **[Important, confirmed]** any HTTPS `licenseurl` is treated as open; **[Minor, confirmed]** one malformed progress entry discards the full persisted map.

## High-level view

The transport DTOs and mapper are never composed behind `CatalogSource`. **[Important, confirmed]** `MovieRepository` therefore has no functioning Internet Archive provider for remote featured, search, or detail behavior.

`InternetArchiveClient` wraps blocking OkHttp `execute()` in `suspend` methods and turns HTTP, I/O, and parsing failures into empty DTOs. **[Important, confirmed]** Callers can block their dispatcher and cannot distinguish no matches from offline/server failure.

`MovieRepository` waits for every source and catches `CancellationException` through broad `Exception` handlers. **[Important, confirmed]** Local results are not publishable immediately, and stale request cancellation is unreliable.

The mapper requires `licenseurl:*` and HTTPS, but HTTPS is not proof of an approved open-content license. **[Important, confirmed]** Arbitrary HTTPS URLs pass and receive the label ‚ÄúGi·∫•y ph√©p m·ªü.‚Äù

**[Minor, confirmed]** Progress restoration is all-or-nothing, so one structurally malformed entry removes valid sibling entries.

<details><summary>Details</summary>

## Internet Archive is not reachable through `CatalogSource`

The only production references to `InternetArchiveClient` and `InternetArchiveMapper` are their declarations; the mapper is instantiated only in its unit test. **[Important, confirmed]** No production class calls `client.search()`/`client.metadata()`, maps the DTOs, and exposes `featured()`, `search()`, and `details()`. The repository can merge curated or test sources but cannot deliver the planned Internet Archive catalog.

Add an Archive-backed `CatalogSource` that owns the client and mapper, maps search results, resolves metadata for details, and defines `featured()`. A MockWebServer test should exercise this adapter through `CatalogSource` and then `MovieRepository`; separate client and mapper tests cannot detect the missing connection.

## Archive requests block callers and erase failure state

`getBody()` calls `httpClient.newCall(request).execute()` directly at `InternetArchiveClient.kt:94‚Äì100`. **[Important, confirmed]** No I/O context or async bridge prevents a ViewModel caller from blocking its dispatcher, and coroutine cancellation is not linked to `Call.cancel()`.

The method also catches every transport outcome, returns `null` for non-2xx, and public methods convert that to empty DTOs. **[Important, confirmed]** Offline, timeout, 500, malformed payload, and a genuine empty result become identical, preventing the planned offline/retry state. Use cancellable OkHttp integration or explicit I/O isolation, rethrow cancellation, and retain a typed failure signal instead of empty success.

## Sequential merging defeats curated-first search and cancellation

`mergeFromSources` invokes sources sequentially and returns after the final source. **[Important, confirmed]** Even with curated data first, `search()` withholds local accent-insensitive matches while a remote source is slow. The current test uses an immediately throwing fake, so it does not verify the ‚Äúlocal search is immediate‚Äù requirement.

Both `details()` and `mergeFromSources()` catch `Exception` at `MovieRepository.kt:19‚Äì23` and `32‚Äì36`. **[Important, confirmed]** This includes `CancellationException`, turning stale-request cancellation into `null`/empty results and allowing later sources to continue. Rethrow cancellation and separate local publication from remote enrichment, using distinct results or a flow; add delayed-source tests for early local emission and prompt cancellation.

## HTTPS-only `licenseurl` is not licensed-only content

The mapper accepts a nonblank HTTPS `licenseurl` at `InternetArchiveMapper.kt:21` and `61`. **[Important, confirmed]** `https://example.com/copyright`, a proprietary EULA, or an unrelated page passes and receives the fallback ‚ÄúGi·∫•y ph√©p m·ªü‚Äù label. The search filter only proves that the field exists.

Validate against accepted Creative Commons and public-domain URI families, normalize known links only if policy permits, and reject unknown authorities/types. Tests should cover valid CC/public-domain URLs, arbitrary HTTPS URLs, and deceptive host/path combinations.

## One corrupt progress record clears the resume library

`loadProgress()` decodes the full object as `Map<String, PlaybackProgress>` and falls back to `emptyMap()` on any exception. **[Minor, confirmed]** One entry with a missing field, wrong type, or incompatible future schema removes every valid resume position. Parse entries independently or through tolerant intermediate DTOs, then filter invalid records; test reconstruction with valid and malformed siblings.

## Boundary behavior is absent from the JVM suite

The fresh run executed 27 tests with zero failures and compiled debug Kotlin. **[Important, confirmed]** No test covers the production Archive adapter, delayed source response, coroutine cancellation, arbitrary-HTTPS license rejection, observable HTTP failure, or mixed valid/malformed progress restoration. Adding those cases would directly exercise all seven findings rather than only the isolated unit happy paths.
